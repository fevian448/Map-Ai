# MapAi — Agent Customizations

This file helps AI coding agents be immediately productive in the MapAi codebase by documenting project-specific conventions, terminal workflows, and key patterns.

## Project Overview

**MapAi** is an Android navigation app (Waze alternative) built with:
- **Frontend:** Kotlin + Jetpack Compose (Android 24+)
- **Backend:** Node.js + Express + better-sqlite3
- **Real-time:** Socket.IO for live data push
- **Maps:** OpenStreetMap (osmdroid) + OSRM/Google Directions API
- **Deployment:** GitHub Pages (web live view), Render/Railway/Docker (backend)

## Terminal Workflows & Build Commands

### Primary Commands

**All terminal workflows run from project root** (`/home/tukuk/AndroidStudioProjects/MapAi/`):

```bash
# Main workflow script (./mapai.sh or ./workflow.sh are equivalent)
./workflow.sh [command]

# Available commands:
./workflow.sh status      # Show git branch, commits, build status
./workflow.sh build       # Compile Android debug APK via Gradle
./workflow.sh push        # Commit and push to GitHub (fevian448/Map-Ai)
./workflow.sh deploy      # Show backend deployment options
./workflow.sh live        # Start backend server locally (Node.js)
./workflow.sh wiki        # Push docs to GitHub Wiki (MapAi-wiki repo)
./workflow.sh all         # Full workflow: build → push → wiki
./workflow.sh monitor     # Open GitHub Actions in browser
./workflow.sh pages       # Open live web view in browser
```

### Configuration Commands

```bash
./config.sh [subcommand]    # Setup AdMob IDs, keystores, environments
./config.sh setup           # Initialize all configurations
./config.sh show            # Display current configuration
./config.sh generate-keystore # Create release signing key
```

### Backend Deployment

Backend runs on Node.js and can deploy to:
1. **Render** (free tier) — Most straightforward
2. **Railway** — CLI-based deployment
3. **Docker** — Self-hosted containers
4. **Local** — `./workflow.sh live` for development

Backend server structure:
- Entry point: `backend/server.js`
- Package: `backend/package.json`
- Database: `backend/data/uploads/` (SQLite WAL)
- Web UI: `backend/public/index.html` (Leaflet.js map viewer)

### Android Build System

```bash
./gradlew clean             # Clean build artifacts
./gradlew assembleDebug     # Build debug APK
./gradlew assembleRelease   # Build release APK (requires keystore)
./gradlew test              # Run unit tests
```

**Build config:** [app/build.gradle.kts](app/build.gradle.kts)
- Target: Android 36 (API 36)
- Min: Android 24 (API 24)
- Language: Kotlin (with Compose)
- Build features: Compose, buildConfig

## Development Conventions

### Language & Styling

- **Kotlin:** Follow Jetpack Compose best practices; use Material 3 components
- **JavaScript/Node.js:** Use Express middleware chain, async/await patterns
- **Bash:** Scripts use set -e (exit on error), colored logging (info/ok/warn/err functions)

### API & Data Flow

- **REST endpoints** under `/api/*` in `backend/server.js`
- **Socket.IO events** for real-time updates (e.g., reports, locations)
- **Models** in Android: [data/Models.kt](app/src/main/java/com/example/mapai/data/Models.kt)
- **DTOs** in backend: inline in `server.js` or config files

### Environment & Secrets

- **AdMob IDs:** Generated by `./config.sh` → `admob_config.properties`
- **Google Maps API Key:** Optional in `local.properties` (fallback to OSRM)
- **Keystore:** Generated by `./config.sh generate-keystore` → `mapai-release.keystore`
- **Backend URLs:** Configurable in Android Settings screen (stored in SharedPreferences)

### Testing & Debugging

- **Android tests:** `app/src/androidTest/` (AndroidJUnitRunner)
- **Unit tests:** `app/src/test/`
- **Local backend:** `./workflow.sh live` starts Express on `http://localhost:3000`
- **Live web:** Browser at `http://localhost:3000` shows Leaflet map with Socket.IO client

## Common Tasks & Patterns

### Building & Deploying

1. **Build Android APK:**
   ```bash
   ./workflow.sh build
   ```
   Output: `app/build/outputs/apk/debug/app-debug.apk`

2. **Deploy backend to Render:**
   - Fork `Map-Ai` repo to your GitHub
   - Run: `./deploy.sh`
   - Follow Render dashboard setup

3. **Push code to GitHub:**
   ```bash
   ./workflow.sh push
   ```

### Configuration

- **Backend URL:** Android Settings → "Server URL" (defaults to `http://10.0.2.2:3000`)
- **Map source:** Settings → select OpenStreetMap or Google
- **Language:** Settings → choose ar/en/es or device locale
- **Emergency contacts:** Settings → add contact numbers for SOS

### Feature Development

- **New API endpoint:** Add route in `backend/server.js`, update `data/remote/MapAiApi.kt` Retrofit interface
- **New screen:** Create folder in `ui/`, add Composable + ViewModel integration
- **New real-time event:** Define Socket.IO listener in `data/remote/SocketManager.kt`
- **Database schema change:** Update SQLite3 init in `backend/server.js` (WAL mode enabled)

## Troubleshooting

| Issue | Solution |
|-------|----------|
| Gradle sync fails | Run `./gradlew clean` and retry Android Studio sync |
| Backend won't start | Check `backend/package.json` dependencies: `cd backend && npm install` |
| GitHub Pages 404 | Enable "GitHub Actions" as source in Settings → Pages; wait 1-2 minutes |
| Android can't connect to backend | Verify backend URL in Settings (use `http://10.0.2.2:3000` for emulator) |
| Wiki push fails | Activate wiki at https://github.com/fevian448/Map-Ai/wiki first |
